# Phase 04.1 Plan 02: Per-Row Stepper + Mouse-Only Shop Summary

One-liner: Rewrote `ShopPanel` input/layout to Stardew-style per-row `-`/qty/`+` steppers with per-row Buy/Sell actions and full click-path logging; removed all keyboard navigation (Escape only) and the persistent-selection machinery.

## What Changed

### New per-row qty state (Task 1)
- `_rowQty[]` (absolute-row-indexed), `_rowQtyTab`, `_rowQtyRowCount`.
- `_qtyMinusRects[]` / `_qtyLabelRects[]` / `_qtyPlusRects[]` per-visible-row arrays (mirror `_rowRects`).
- `EnsureRowQtySized(int rowCount)` — reallocates + fills 1 on tab switch or row-count change.
- `GetMaxQty(rowIndex)` — Buy: `min(gold/price, stack-headroom)`; Sell: `stack.Quantity`.
- `ComputeStackHeadroom(itemId, stackLimit)` — total additional units inventory can accept.

### New TryBuy/TrySell signatures (Task 1)
- `TryBuy(int rowIndex, int qty, out ToastRequest?)` — validates gold & headroom × qty; partial-refunds if `TryAdd` returns leftover.
- `TrySell(int rowIndex, int qty, out ToastRequest?)` — clamps qty to `stack.Quantity`; calls `EnsureRowQtySized(GetRowCount())` after sale.

### Mouse-only Update (Task 2)
Update() now does only: wheel scroll → layout → hover hit-test → on click: close-X → click-outside → Buy tab → Sell tab → per-row `qty-` / `qty+` / action → Escape key.

### Full logging catalogue
Every click path emits a `[ShopPanel]` log:

| Path | Log |
|---|---|
| Close X clicked | `[ShopPanel] Close X clicked` |
| Click outside panel | `[ShopPanel] Click outside panel -> close` |
| Buy tab change | `[ShopPanel] Tab -> Buy` |
| Sell tab change | `[ShopPanel] Tab -> Sell` |
| qty- pressed | `[ShopPanel] qty- row={abs} qty={new}` |
| qty+ pressed | `[ShopPanel] qty+ row={abs} qty={new} cap={cap}` |
| Buy pressed (enabled) | `[ShopPanel] Buy-click row={abs} qty={q}` |
| Sell pressed (enabled) | `[ShopPanel] Sell-click row={abs} qty={q}` |
| action on disabled row | `[ShopPanel] action-click row={abs} disabled` |
| Escape pressed | `[ShopPanel] Escape pressed -> close` |

Transaction-internal logs (`Bought ...`, `Sold ...`) preserved from prior wave.

### Removals
- Keyboard handlers: `Keys.Up`, `Keys.Down`, `Keys.Left`, `Keys.Right`, `Keys.Tab`, `Keys.Enter` — all gone. Only `Keys.Escape` remains.
- Fields: `_selectedIndex`, `_sellQty`, `_lastTabForQty`, `_lastSelectedIndexForQty`, singleton `_qtyMinusRect` / `_qtyLabelRect` / `_qtyPlusRect`, `SelectBg`.
- Methods: `CurrentSellStack`, `ComputeDisabledReason`, `DrawDisabledReason`.
- Constants: `ReasonNotEnoughGold`, `ReasonInventoryFull`, `ReasonSelectToSell`, `ReasonCannotSell`.
- `DrawRow` `bool selected` parameter (no selection concept) and the selection highlight it drew.

## Deviations from Plan

None material. Minor adjustments:
- Renamed local `label` to `qtyLabel` inside per-row stepper draw to avoid conflict with an outer-scope `label` in `DrawRow` (C# error CS0136). Fixed during Task 1 build iteration.
- Plan left open whether to keep `_selectedIndex` as a dead field or remove it. I removed it entirely along with `CurrentSellStack`, `SelectBg`, and the stale row/scroll clamp in `TrySell` — there are no remaining readers. Only a rationale comment in `UpdateLayoutCache` mentions the name (text, not code).

## Decisions Made

- **Full-remove selection machinery**: `_selectedIndex`, `CurrentSellStack`, `SelectBg`, and the `_selectedIndex` clamp in `TrySell` are all gone. Cleaner surface for future plans.
- **Per-row qty reset after transaction**: After a successful Buy/Sell click, the row's qty resets to 1 (same row may still be visible after Buy; row may have vanished after Sell — guard with bounds check).
- **Buy partial success path**: If `_inv.TryAdd(qty)` leaves `leftover > 0` (shouldn't happen after headroom check, but safety), refund `entry.Price * leftover` rather than the whole transaction. User still gets the portion that fit.
