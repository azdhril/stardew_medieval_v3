# Phase 04.1 Plan 03: Save Round-Trip Summary

One-liner: Introduced `GameStateSnapshot.SaveNow(Services)` as the single save entry point and wired it into FarmScene.OnDayAdvanced, ShopOverlayScene close, and a new GameplayScene F5 hotkey so shop gold, quest advancement, and inventory changes persist regardless of which scene the player quits from.

## What Was Built

### Task 1 — `GameStateSnapshot.SaveNow` helper
New static class `stardew_medieval_v3.Core.GameStateSnapshot` with one public method:

```csharp
public static void SaveNow(ServiceContainer services, List<FarmCellSaveData>? farmCells = null)
```

Behavior:
- Builds a `GameState` DTO from live services (`Time`, `Player`, `Inventory`, `Quest`).
- Falls back to the prior `services.GameState` for fields that have no live owner on a non-Farm save (e.g. `FarmCells`, `CurrentScene`, `BossKilled`, or stamina/position when `Player` is null).
- Calls `services.Inventory?.SaveToState(state)` and `services.Quest?.SaveToState(state)`.
- Logs `[GameStateSnapshot] SaveNow: day=... gold=... quest=... items=... scene=...` for each flush.
- Delegates the actual write to `SaveManager.Save(state)`.

Commit: `f4a75d8`

### Task 2 — Wire SaveNow at three pinch points
1. **FarmScene.OnDayAdvanced** — replaced inline `new GameState { ... } + two SaveToState calls + SaveManager.Save` with `GameStateSnapshot.SaveNow(Services, _gridManager.GetSaveData())`. BossKilled/CurrentScene flushed into `Services.GameState` before the call so the helper can propagate them.
2. **ShopOverlayScene.Update** — on close path (`_panel.Update` returns true), call `GameStateSnapshot.SaveNow(Services)` before `PopImmediate`. Log tag changed to `[ShopOverlayScene] Closed -> SaveNow`.
3. **GameplayScene.Update** — added a global `Keys.F5` handler at the top of the shared update path (before Escape/Inventory/etc.) so every gameplay scene (Farm/Village/Castle/Shop) inherits manual save. Logs `[{SceneName}Scene] F5 manual save`.

Commit: `04c484b`

## Round-Trip Contract (verification design)

The log lines below, emitted by the new helper, constitute the round-trip proof surface. After any save pinch-point fires, `savegame.json` at `%LOCALAPPDATA%/StardewMedieval/savegame.json` must reflect the values in the last `[GameStateSnapshot] SaveNow` line.

| Trigger | Expected log | JSON fields that MUST match live |
| --- | --- | --- |
| Sleep P (Farm) | `[FarmScene] === Day N ===` then `[GameStateSnapshot] SaveNow: day=N ...` | DayNumber, Stamina, FarmCells, Gold, Inventory, QuestState, Equipment, Hotbar |
| Shop close (Esc/X/click-outside) | `[ShopOverlayScene] Closed -> SaveNow` + `[GameStateSnapshot] SaveNow: ...` | Gold, Inventory (post-transaction) |
| F5 in any gameplay scene | `[{Scene}Scene] F5 manual save` + `[GameStateSnapshot] SaveNow: ...` | Every live field (full snapshot) |

No JSON diff is captured programmatically — the manual UAT step listed in the plan's verification block is the acceptance gate. The code is structured so any subsequent diff tool only needs to read `savegame.json` before and after a `SaveNow` log line.
