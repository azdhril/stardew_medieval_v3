<objective>
Fix the scroll-wheel regression in `ShopPanel`. Today, `UpdateLayoutCache` runs follow-selection every frame and clamps `_scrollOffset` back toward `_selectedIndex`, so user wheel scrolling is reset on the next frame. The fix: follow-selection must only activate when `_selectedIndex` actually changed (keyboard navigation), not on every Update tick.

Purpose: UAT gap "Scroll wheel do mouse deve rolar a lista de itens do shop" — currently fails because follow-selection fights user input.
Output: `_scrollOffset` is user-controllable via wheel without being yanked back; keyboard navigation still auto-scrolls to keep selection visible.
</objective>
