<objective>
Close the Test 6 major gap from 05-UAT.md: dungeon chests currently re-seed with fresh loot and re-close on every new run, so dying and re-entering is infinite loot farming. Chest collection must be a one-time event per save file, matching the village/farm chest semantics.

Purpose: Opened chests are the dungeon's treasure gate. Allowing them to refill on death breaks the game's economy (the player can farm any chest infinitely by suiciding). This is the last UAT blocker for Phase 5.

Output:
- `DungeonState.BeginRun()` no longer clears `OpenedChestIds`. It still clears `ClearedRooms`, regenerates `RunSeed`, and clears `ChestContents` — but chest *opened* state persists across runs.
- `DungeonChestSeeder.Seed()` skips any chest whose id is in `OpenedChestIds` (writes empty content list for it instead of rolling the loot table). Unopened chests roll as before.
- `DungeonScene.OnPreUpdate()` death branch calls `GameStateSnapshot.SaveNow(Services)` BEFORE `BeginRun()` so the `OpenedChestIds` list is flushed to disk even if the game is killed after the wipe.
- `VillageScene.BeginDungeonRun` unchanged in sequencing — `BeginRun` then `Seed` — but now both correctly honor the persisted opened-chest set.
- Re-entering a dungeon room after death shows the *previously-opened* chest in its "opened + empty" state (same code path as existing re-entry, which already respects `IsChestOpened`).
</objective>

```csharp
// Death -> reset run + transition back to farm.
if (!Player.IsAlive)
{
    Console.WriteLine($"[DungeonScene:{_room.Id}] Player died -- resetting run");
    // Persist BEFORE wiping: opened chests now survive BeginRun in memory, but we
    // must also flush them to disk so a crash-after-death does not forget them.
    // (05-UAT Gap 2 hardening — belt-and-suspenders over the in-memory fix.)
    GameStateSnapshot.SaveNow(Services);
    Services.Dungeon!.BeginRun();
    DungeonChestSeeder.Seed(Services);
    Player.HP = Player.MaxHP;
    Services.SceneManager.TransitionTo(new FarmScene(Services, "DungeonDeath"));
    return true;
}
```

**Why save BEFORE BeginRun even though BeginRun preserves OpenedChestIds now:**
The in-memory fix (Task 1) means opened chests survive a normal play session. But if the player dies and then quits (Alt+F4 / power loss) before the next scheduled autosave, the opened-chest set would be lost with the process. Saving at the moment of death is the cheapest way to make chest persistence identical to the boss-defeat flag (which already persists via mid-combat `GameStateSnapshot.SaveNow` in the boss victory handler). A single extra `SaveNow` on death is negligible I/O.

**No other changes to DungeonScene.cs in this task.** Do not touch OnLoad hydration, chest open flow, or the boss handler — they are already correct and hitting `GameStateSnapshot.SaveNow` elsewhere for their own reasons.
