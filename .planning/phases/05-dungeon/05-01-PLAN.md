<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Disk → Save loader | `savegame.json` may be hand-edited or corrupted by user; untrusted JSON enters `SaveManager.Load`. |
| Disk → TMX loader | TMX files under `assets/Maps/` treated as trusted (shipped), but `TileMap.Load` is the parse surface. |
| Test host → Production code | xunit process loads the same assembly; no boundary crossed at runtime. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | `SaveManager.Load` reading v7/v8 JSON | mitigate | Version gate in `MigrateIfNeeded`; wrap JSON parse in try/catch; if migration throws, fall back to new-game path (existing pattern). Add v7→v8 migration test to lock behavior. |
| T-05-02 | Denial of Service | `GameStateSnapshot.SaveNow` writing chest + dungeon state | accept | Local single-player; save file write failure only risks progress loss which is already handled by existing IOException catch in SaveManager. |
| T-05-03 | Information Disclosure | Save file contains inventory/quest state | accept | Local-only file under `%LOCALAPPDATA%`; no PII; no network surface. |
| T-05-04 | Tampering | Hand-edited DungeonState in save (e.g., `BossDefeated=true` bypass) | accept | Single-player local game; cheating is the user's own save. No integrity check needed for v1. |
</threat_model>
