## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Hardcoded 960x540 in UI overlays | DialogueBox AND ShopPanel both contained `ScreenWidth = 960` consts plus `new Rectangle(0, 0, 960, 540)` dim overlays. In any non-960x540 viewport (especially fullscreen) the dim does not cover the screen and the panel sits in the wrong position. DialogueBox was fixed in 05-05 but ShopPanel was missed because it was not in the plan scope. | advisory | Before declaring viewport-awareness work complete, grep `src/UI/` and `src/Scenes/` for `960` and `540` literals AND `ScreenWidth` / `ScreenHeight` consts; every overlay scene must thread viewport into Draw. |
| Camera desync when viewport changes mid-overlay | F11 fullscreen toggle in Game1 calls `Camera.FitZoomToViewport` which updates Zoom but NOT Position. The paused gameplay scene under DialogueScene/ShopOverlayScene does NOT receive Update (SceneManager only updates the top scene), so `Camera.Follow()` never re-clamps Position against the new viewport/zoom. Result: tiles or player look misaligned until the overlay closes. | advisory | After any viewport change (fullscreen toggle, window resize), call the new `Camera.Reclamp()` so paused scenes underneath stay correctly framed. |
