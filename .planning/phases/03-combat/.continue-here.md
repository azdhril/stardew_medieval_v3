## Critical Anti-Patterns (do NOT repeat these)
- [ANTI-PATTERN] Using CollisionBox (feet) for combat damage checks → projectiles/attacks feel unfair because player body isn't the target. **Mitigation**: Always use HitBox for combat; CollisionBox is strictly for terrain/movement.
- [ANTI-PATTERN] Relying on body-overlap (HitBox.Intersects) for enemy melee when AI stops at AttackRange > 0 → enemies freeze just out of reach, never hit. **Mitigation**: Enemies that attack melee need a projected reach hitbox (GetMeleeAttackHitbox / GetBossSlashHitbox), not body overlap.
- [ANTI-PATTERN] Hardcoded facing direction in attack hitboxes (boss originally faced Down only) → attack only works in one direction. **Mitigation**: Compute facing from `_lastPlayerPos - Position` dominant axis each attack.
- [ANTI-PATTERN] Visual (white slash) geometry drifting from hitbox (red) geometry → player distrusts feedback. **Mitigation**: SlashEffect must mirror MeleeAttack dimensions/offsets exactly.
