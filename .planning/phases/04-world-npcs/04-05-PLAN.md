<objective>
Close the BLOCKER gap from UAT Test 13: after F1 + relaunch, Gold / quest state / inventory are lost. Root cause (see DEBUG-save-persistence.md): `FarmScene.OnLoad` unconditionally creates a fresh `InventoryManager` and `MainQuest` on every scene re-entry and hydrates them from the last on-disk save — wiping any in-memory changes the player made in Village/Castle/Shop.

Purpose: Mirror the existing `Services.Player == null` guard (FarmScene.cs:64) onto `Services.Inventory` and `Services.Quest`. Live instances are created once on first Farm entry and reused across Farm re-entries. The disk hydration only runs on first creation. This narrowly fixes the blocker without touching save format, migration, or any other scene.

Output: A single-file edit to `src/Scenes/FarmScene.cs` adding two null-check guards and scoping save hydration + starter seeding behind them. No new classes, no schema changes.
</objective>
