## Root Cause Detail

`ShopPanel.Update` (line 93-98):
```csharp
int wheel = input.ScrollWheelDelta;
if (wheel != 0)
{
    int maxScroll = Math.Max(0, rows - VisibleRows);
    _scrollOffset = Math.Clamp(_scrollOffset - Math.Sign(wheel), 0, maxScroll);
}
```

Immediately after, `UpdateLayoutCache` (line 218-220):
```csharp
if (_selectedIndex < _scrollOffset) _scrollOffset = _selectedIndex;
else if (_selectedIndex >= _scrollOffset + VisibleRows) _scrollOffset = _selectedIndex - VisibleRows + 1;
```

When `_selectedIndex = 0` and wheel scrolls to `_scrollOffset = 1`, first branch fires → `_scrollOffset = 0`. Scroll reset same frame. Arrow keys work because they change `_selectedIndex` first, and follow-selection then pushes scroll along.

## Proposed Fix (3 lines)

Replace lines 93-98 of `src/UI/ShopPanel.cs` with:

```csharp
int wheel = input.ScrollWheelDelta;
if (wheel != 0)
{
    int maxScroll = Math.Max(0, rows - VisibleRows);
    _scrollOffset = Math.Clamp(_scrollOffset - Math.Sign(wheel), 0, maxScroll);
    // Pull selection into the new viewport so follow-selection in UpdateLayoutCache doesn't fight back.
    if (_selectedIndex < _scrollOffset) _selectedIndex = _scrollOffset;
    else if (_selectedIndex >= _scrollOffset + VisibleRows) _selectedIndex = _scrollOffset + VisibleRows - 1;
}
```
